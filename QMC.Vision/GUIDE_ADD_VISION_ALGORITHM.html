새 비전 알고리즘 추가 가이드

초보자도 따라 할 수 있게 — “이 파일 열고, 이거 복붙하고, 빌드” 순서 · 장비PC(V:) 코드 기준

먼저 한 가지만 이해하세요. 비전 알고리즘은 딱 두 종류입니다.
• Finder = 패턴을 찾기/정렬 (Reticle, Die, EjectPin…) → IPatternFinder 구현
• Inspector = 합/불 검사 (표면, 안착 갭, 치핑…) → IInspector 구현
추가 작업은 둘 다 똑같은 5단계입니다. 아래는 Inspector 추가 기준이고, 끝에 Finder 차이만 정리했습니다.
실제 예시가 이미 있습니다. PlacementGapInspector(빈 비전 안착 검사)가 정확히 이 5단계로 추가된 알고리즘입니다. 막히면 그 파일을 그대로 참고하세요.

전체 5단계 한눈에

1

알고리즘 클래스 작성

Equipment/Core/MyInspector.cs 를 새로 만들고 IInspector를 구현합니다. 계약은 딱 3개(Id, InspectionRoi, Inspect())뿐입니다.

using System.Drawing;
namespace QMC.Vision.Core
{
    /// <summary>내 검사 알고리즘 — 한 줄 설명.</summary>
    public class MyInspector : IInspector
    {
        public string Id { get; }
        public Roi InspectionRoi { get; set; }

        // ▼ 화면에서 조정할 파라미터 (4단계에서 레시피와 연결)
        public double Threshold { get; set; } = 128;

        public MyInspector(string id)
        {
            Id = id;
            InspectionRoi = new Roi { Name = id + ".Roi", CenterX = 320, CenterY = 240, Width = 400, Height = 300 };
        }

        public InspectionResult Inspect(Bitmap image)
        {
            var res = new InspectionResult { RoiName = Id };
            if (image == null) { res.IsPass = false; res.ErrorMessage = "no image"; return res; }
            try
            {
                // === 여기에 실제 이미지 처리 로직 작성 ===
                double measured = 0.0;        // 측정값 계산
                bool pass = measured <= Threshold;

                res.IsPass = pass;
                res.Items.Add(new InspectionItem { Name = "MyValue", Value = measured.ToString("F3"), IsPass = pass });
                return res;
            }
            catch (System.Exception ex)
            {
                res.IsPass = false; res.ErrorMessage = "my inspect: " + ex.Message; return res;
            }
        }
    }
}
결과 규칙: IsPass로 합/불, Items에 측정 항목(이름·값·합불)을 담고, 에러는 ErrorMessage. 시퀀서/TCP가 이 결과를 그대로 PASS/FAIL로 해석합니다.

이미지 → 그레이 배열 변환 같은 헬퍼가 필요하면 PlacementGapInspector.ToGray()(LockBits 고속)를 복사해 쓰세요.

2

백엔드 팩토리에서 생성 연결

모듈은 Backend.CreateInspector(id)로 검사기를 만듭니다. 여기에 내 알고리즘을 고를 분기를 추가합니다(id 이름으로 구분).

Equipment/Backends/Cognex/CognexBackend.cs 
public IInspector CreateInspector(string id)
{
    if (id != null && id.IndexOf("Placement", StringComparison.OrdinalIgnoreCase) >= 0)
        return new PlacementGapInspector(id);
    // ▼ 추가: id 에 "My" 가 들어가면 내 검사기 생성
    if (id != null && id.IndexOf("My", StringComparison.OrdinalIgnoreCase) >= 0)
        return new MyInspector(id);

    return new CognexInspector(id, this);   // 기본
}
주의: 백엔드는 Cognex/OpenCv/Sim 3개입니다. 백엔드 무관 알고리즘(Bitmap 직접 처리)이면 Sim 백엔드에도 같은 분기를 넣어야 Sim 모드에서 테스트됩니다. (Backends/Sim/SimBackend.cs)
3

모듈에 등록

검사기를 쓸 모듈(예: BinVision)에서 AddInspector로 등록합니다. id가 위 팩토리 분기와 일치하면 내 알고리즘이 붙습니다.

Equipment/Unit/BinVisionModule.cs 
public IInspector MyInspector { get; }   // 속성 추가(선택)

public BinVisionModule(ICamera camera, IVisionBackend backend)
    : base("BinVision", camera, backend)
{
    // 기존 …
    MyInspector = AddInspector<InspectorAlgoSetup, InspectorAlgoConfig, InspectorAlgoRecipe>("MyInspector");
}

AddInspector가 내부에서 Backend.CreateInspector("BinVision/MyInspector")를 호출 → 2단계 분기로 MyInspector 생성 → 노드 등록까지 자동.

4

레시피 파라미터 추가 (화면 노출)

사용자가 화면에서 임계값을 바꾸게 하려면, 레시피에 필드를 넣고 노드가 레시피→알고리즘으로 주입하게 합니다.

4-a. 레시피에 필드 추가

Equipment/Unit/AlgorithmData.cs → InspectorAlgoRecipe 
public class InspectorAlgoRecipe : AlgoRecipeBase
{
    [DataMember] public Roi InspectionRoi { get; set; }
    [DataMember] public double Threshold { get; set; }
    // ▼ 내 알고리즘 전용 파라미터가 더 필요하면 추가
    [DataMember] public double MyParam { get; set; }
    // SetDefaults() 에 기본값도 추가
}

4-b. 노드가 주입하도록 연결

Equipment/Unit/AlgorithmNode.cs → ApplyToRuntime() / CollectFromRuntime() 
// ApplyToRuntime: 레시피 → 알고리즘
else if (_inspector is QMC.Vision.Core.MyInspector mi)
{
    mi.Threshold = r.Threshold;
    // mi.MyParam = r.MyParam;
}
// CollectFromRuntime: 알고리즘 → 레시피 (저장용, 반대로 1줄씩)
화면 표시는 자동입니다. 레시피 그리드가 InspectorAlgoRecipe의 필드를 읽어 파라미터 칸을 만듭니다. (전용·복잡 파라미터는 IAlgoParamSync 인터페이스를 알고리즘에 구현하면 노드가 자동 호출 — ApplyParams/CollectParams.)
5

검사 순서에 단계 추가

그 모듈의 사이클에서 언제 검사할지 CycleSteps()에 한 줄 넣습니다.

Sequencing/BinVision/BinVisionSequence.cs 
protected override IEnumerable<KeyValuePair<string,string>> CycleSteps()
{
    yield return Step("GRAB", null);
    yield return Step("MATCH",   "ReticleFinder");
    yield return Step("INSPECT", "PlacementInspector");
    yield return Step("INSPECT", "MyInspector");   // ◀ 추가
}

시퀀서는 모듈에 없는 id면 자동 skip하므로 안전합니다. (순서를 화면에서 바꾸고 싶으면 별도 "데이터 기반 CycleSteps" 작업이 필요 — 문의)

+

프로젝트 등록 + 빌드

새 .cs 파일은 구식 csproj라 자동 포함이 안 됩니다. 한 줄 등록하세요.

QMC.Vision.csproj 
<Compile Include="Equipment\Core\MyInspector.cs" />

그 후 빌드 → 레시피 화면에서 MyInspector 선택 → ROI·임계값 셋팅·저장 → RUN.

Finder(패턴 찾기)를 추가할 땐 (차이만)

동일한 5단계, 인터페이스만 IPatternFinder:

  • 구현할 멤버: Id, SearchRoi, TrainRoi, TrainImage, AcceptThreshold, MaxInstances, Train(Bitmap), LoadTrainImage(Bitmap), Match(Bitmap)→MatchResult
  • 2단계는 CreatePatternFinder 분기, 3단계는 AddFinder<…>("MyFinder"), 5단계는 Step("MATCH","MyFinder")
  • 레시피는 FinderAlgoRecipe(SearchRoi/TrainRoi/AcceptThreshold/TrainModelPath…)
  • 학습(Train)이 필수 — Match 전에 패턴이 있어야 함. 화면 TRAIN 버튼이 Train() 호출.

완료 체크리스트

  • 1. IInspector/IPatternFinder 구현 클래스 작성 (Equipment/Core)
  • 2. 백엔드 팩토리(CreateInspector/CreatePatternFinder)에 id 분기 추가 (Sim 백엔드도 필요 시)
  • 3. 모듈 생성자에서 AddInspector/AddFinder 등록
  • 4. InspectorAlgoRecipe/FinderAlgoRecipe에 파라미터 + AlgorithmNode 주입(Apply/Collect)
  • 5. CycleSteps()에 단계 추가
  • +. csproj에 <Compile Include> 등록
  • 빌드 → 레시피 화면 셋팅·저장 → RUN 확인 (Sim 백엔드로 먼저 테스트 권장)

초보가 자주 막히는 곳

증상원인 / 해결
빌드는 됐는데 내 알고리즘이 안 불림2단계 팩토리 분기의 id 키워드와 3단계 AddInspector("…") id가 안 맞음. 같은 단어 포함되게.
파일이 빌드에 안 들어감csproj <Compile Include> 누락(구식 csproj).
화면에서 바꾼 값이 검사에 반영 안 됨4단계 ApplyToRuntime 주입 누락. 레시피→알고리즘 1줄 확인.
Sim 모드에선 동작 안 함Sim 백엔드 팩토리에 분기 누락. 백엔드별로 추가.
판정값은 맞는데 합·불이 이상단위(px/mm)·임계값 점검. (PlacementGap의 PixelSize 이슈 참고)
요약. 알고리즘 로직(이미지 처리)은 코드로 작성하지만, 붙이는 절차는 5단계 고정 패턴입니다 — 클래스 작성 → 팩토리 분기 → 모듈 등록 → 레시피 파라미터 → 시퀀스 단계 (+csproj). PlacementGapInspector를 옆에 띄워놓고 똑같이 따라 하면 됩니다.
