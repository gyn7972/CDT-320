QMC.Vision 시퀀서 — 코드 구현 방식

어떤 코드를 써서 RUN → 실제 검사 실행까지 가는가 (장비PC V: 소스 기준) · 네임스페이스 QMC.Vision.Sequencing

한 줄 원리. UI 버튼이 호스트에 위임 → 호스트가 컨텍스트+조정자를 만들고 → 조정자가 선택 모듈마다 시퀀스를 Task.Run으로 병렬 실행 → 각 시퀀스는 async/await 루프로 사이클을 돌며 디스패처를 거쳐 공유 명령 코어를 호출합니다. 정지는 CancellationToken, 수동 진행은 SemaphoreSlim 게이트로 구현돼 있습니다.

3Host — 컨텍스트와 조정자를 만든다

QMC.Vision/Sequencing/Common/VisionAutoSequenceHost.cs
public void StartModules(SequenceModuleKind kind, SequenceRunMode mode, int cycleIntervalMs){
    Stop();                                  // 기존 실행 정지
    if (kind == SequenceModuleKind.None) return;
    var ctx = new VisionSequenceContext(_machine, null, LogSink){   // 디스패처 null→기본 Direct
        CycleIntervalMs = cycleIntervalMs > 0 ? cycleIntervalMs : 500
    };
    _coordinator = new AutoSequenceCoordinator(ctx);
    _coordinator.Start(kind, mode);
}

public void Start(SequenceModuleKind selected, SequenceRunMode mode){
    Stop();
    _cts = new CancellationTokenSource();
    CancellationToken ct = _cts.Token;
    foreach (var kv in _sequences){
        if ((selected & kv.Key) == 0 || kv.Value == null) continue;  // 비트플래그 선택
        kv.Value.Configure(mode);
        ModuleSequenceBase seq = kv.Value;
        _running.Add(Task.Run(() => seq.RunAsync(ct)));        // ★ 모듈별 병렬 실행
    }
}

핵심: 선택된 모듈들이 각자 별도 Task로 동시에 돌고, 하나의 CancellationToken으로 일괄 정지됩니다. 정지 시 cts.Cancel() + Dispose()(누수 방지).

5Sequence 실행 — Auto 루프 vs Step 게이트

QMC.Vision/Sequencing/Common/ModuleSequenceBase.cs
public async Task RunAsync(CancellationToken ct){
    if (Mode == SequenceRunMode.Auto){
        await ExecuteAutoAsync(ct); return;            // 연속
    }
    while (!ct.IsCancellationRequested){                  // Manual/Step
        await _stepGate.WaitAsync(ct);                     // ★ 신호 올 때까지 대기
        await ExecuteStepAsync(ct);                        // 1사이클
    }
}
// Auto: 사이클 → 간격 대기 무한 반복
protected virtual async Task ExecuteAutoAsync(CancellationToken ct){
    while (!ct.IsCancellationRequested){
        await RunModuleCycleAsync(ct);
        await Task.Delay(CycleIntervalMs, ct);
    }
}
// StepUnit(): UI '한 단계' → 게이트 1회 해제(중복 방지 가드)
public void StepUnit(){
    if (Interlocked.CompareExchange(ref _stepBusyOrQueued, 1, 0) == 0)
        _stepGate.Release();
}

Auto=while+Task.Delay 연속, Step=SemaphoreSlim 게이트(WaitAsync↔Release)로 1단계씩. 둘 다 CancellationToken으로 즉시 취소.

var chipUid = ResolveCycleChipUid();                  // SimEmitChipUid면 합성 발급(자재추적)
foreach (var step in CycleSteps()){                  // (cmd, id) 순서
    string cmd = step.Key, id = step.Value;
    if (cmd == "MATCH"   && !Module.Finders.ContainsKey(id))   continue;  // 없으면 skip
    if (cmd == "INSPECT" && !Module.Inspectors.ContainsKey(id)) continue;

    if (IsStepFail(result)) { /* fail:/ERR → NG + 사람이 읽는 사유 */ }
    var ng = MatchNgIfBelowThreshold(cmd, id, result);    // score < AcceptThreshold → NG
}
// 스톱워치로 LastCycleMs / CycleCount 갱신(메트릭)

7디스패치 → 공유 코어 (실제 명령 실행)

// Context: 예외를 삼키지 않고 ERR: 문자열로 변환
public string Dispatch(IVisionModule module, string cmd, string[] args){
    try { return Dispatcher.Execute(module, cmd, args); }
    catch (Exception ex) { return "ERR:" + ex.Message; }
}
// Direct: TCP 서버와 '동일 코어' 호출 (결과·mm변환·로그 일관)
switch (cmd.ToUpperInvariant()){
    case "GRAB":    return VisionCommandCore.Grab(module);
    case "MATCH":   return VisionCommandCore.Match(module, cfg, id, chipUid);
    case "INSPECT": return VisionCommandCore.Inspect(module, cfg, id, chipUid);
    case "TRAIN":   return VisionCommandCore.Train(module, id);
}

→ 실제 운영의 VisionTcpServer도 같은 VisionCommandCore를 부릅니다. 그래서 Sim 자체실행과 핸들러 TCP 경로의 결과가 동일합니다.

스레딩 · 수명 모델 (요약)

  • 병렬성: 모듈마다 Task.Run(RunAsync) — UI 스레드 비차단. 로그는 BeginInvoke로 UI 마샬링.
  • 정지: 공유 CancellationTokenSource → Cancel()+Dispose() → 모든 await 지점에서 OperationCanceledException으로 빠져나옴.
  • 수동 진행: SemaphoreSlim 게이트. StepUnit()가 Release, 직전 step 진행 중이면 Interlocked 가드로 무시.
  • 모드: Configure(mode)로 Auto/Manual/Step 지정 후 RunAsync가 분기.
  • 판정: fail:/ERR=실행실패, FAIL;=검사NG(정상), MATCH score < AcceptThreshold=NG.
  • 메트릭: 각 사이클 Stopwatch → LastCycleMs/CycleCount → Snapshot() → UI 0.7s 폴링.
정리. “자체 구현 방식” = UI는 호스트 API만 호출 → 호스트가 컨텍스트+조정자 생성 → 조정자가 Task.Run으로 모듈별 시퀀스를 병렬 실행 → 각 시퀀스는 async 루프(또는 SemaphoreSlim 게이트)로 CycleSteps()를 돌며 Context.Dispatch → VisionCommandCore로 실제 명령을 실행. 외부 라이브러리 없이 .NET Task/SemaphoreSlim/CancellationToken만으로 구성돼 있습니다.
