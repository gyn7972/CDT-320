# SP Remote Viewer

TCP/IP 기반 원격 화면 공유 프로그램

## 프로그램 개요

SP Remote Viewer는 한 PC의 화면을 다른 PC에서 실시간으로 볼 수 있는 원격 화면 공유 솔루션입니다.

### 주요 기능

1. **서버 모드 (화면 전송)**
   - 현재 PC의 화면을 캡처하여 클라이언트로 전송
   - **다중 모니터 지원** - 여러 모니터 중 원하는 모니터 선택 가능
   - TCP/IP 서버로 동작
   - JPEG 압축을 통한 데이터 전송 최적화
   - 압축 품질 및 FPS 조절 가능

2. **클라이언트 모드 (화면 보기)**
   - 서버로부터 화면 데이터를 수신하여 표시
   - 커스텀 UI 컨트롤로 구현
   - 화면 크기 자동 조절 옵션

## 사용 방법

### 서버 설정 (화면을 공유하는 PC)

1. **"서버 (화면 전송)"** 탭 선택
2. 설정값 조정:
   - **포트 번호**: 클라이언트가 접속할 포트 (기본값: 8888)
   - **모니터**: 전송할 모니터 선택 (다중 모니터 환경에서 선택 가능)
     - 모니터 1, 2, 3... 형태로 표시
     - 주 모니터는 [주 모니터] 표시
     - 해상도 정보 함께 표시
   - **압축 품질**: JPEG 압축 품질 (1-100, 높을수록 화질 좋음)
   - **FPS**: 초당 전송 프레임 수 (1-60)
3. **"시작"** 버튼 클릭
4. 클라이언트 접속 대기
5. 로그 창에서 연결 상태 확인

### 클라이언트 설정 (화면을 보는 PC)

1. **"클라이언트 (화면 보기)"** 탭 선택
2. 연결 정보 입력:
   - **서버 주소**: 서버 PC의 IP 주소 (로컬 테스트시 127.0.0.1)
   - **포트**: 서버에서 설정한 포트 번호
3. **"연결"** 버튼 클릭
4. 서버 화면이 실시간으로 표시됨
5. **"화면에 맞춤"** 체크박스로 화면 크기 조절 가능

## 기술 사양

### 개발 환경
- **.NET Framework 4.7.2**
- **C# Windows Forms**

### 주요 구성 요소

#### 1. ScreenCaptureServer.cs
- 화면 캡처 및 TCP/IP 서버 기능
- **다중 모니터 지원** (Screen.AllScreens 사용)
- JPEG 압축을 통한 데이터 최적화
- 멀티스레드 기반 실시간 전송

#### 2. RemoteViewerControl.cs
- 커스텀 UI 컨트롤
- TCP/IP 클라이언트 기능
- 더블 버퍼링을 통한 깜빡임 방지
- 실시간 이미지 수신 및 표시

#### 3. Form1.cs
- 서버/클라이언트 UI 관리
- 모니터 선택 콤보박스
- 사용자 인터페이스 제어

### 네트워크 프로토콜

데이터 전송 형식:
```
[4 bytes: 데이터 크기] + [N bytes: JPEG 압축된 이미지 데이터]
```

## 다중 모니터 지원

### 특징
- 시스템에 연결된 모든 모니터 자동 감지
- 드롭다운 메뉴에서 원하는 모니터 선택
- 각 모니터의 해상도 정보 표시
- 주 모니터 자동 표시

### 사용 예시
```
모니터 1 (1920x1080) [주 모니터]
모니터 2 (2560x1440)
모니터 3 (1920x1080)
```

## 성능 최적화 팁

1. **압축 품질 조절**
   - 네트워크 속도가 느린 경우: 품질 30-50
   - 고속 네트워크: 품질 70-90

2. **FPS 조절**
   - 일반 사무 작업: 5-10 FPS
   - 동영상 시청: 15-30 FPS
   - 고성능 요구: 30-60 FPS

3. **네트워크 설정**
   - 방화벽에서 사용하는 포트 허용 필요
   - 외부 네트워크 접속시 포트 포워딩 설정

4. **다중 모니터 환경**
   - 해상도가 높은 모니터는 압축 품질을 낮추거나 FPS를 줄여 대역폭 절약
   - 필요한 모니터만 선택하여 전송

## 주의 사항

- 서버는 한 번에 하나의 클라이언트만 연결 가능
- 대용량 화면 또는 높은 FPS 설정시 네트워크 대역폭 소모 증가
- 고해상도 모니터 전송시 압축 품질과 FPS를 적절히 조절 권장
- 보안이 필요한 경우 별도의 암호화 구현 필요 (현재 미구현)
