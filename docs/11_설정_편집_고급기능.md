## 2. Round 12: AxisSetup APPLY 확장

기존: `SoftLimitMinus / SoftLimitPlus` 만 적용
이후: **DefaultVelocity + Stroke** 도 reflection 으로 적용 (해당 속성 존재 시)

```csharp
var velProp = setupType.GetProperty("DefaultVelocity") ?? setupType.GetProperty("Velocity");
if (velProp != null && velProp.CanWrite) velProp.SetValue(ax.Setup, match.DefaultVel);
var strokeProp = setupType.GetProperty("Stroke");
if (strokeProp != null && strokeProp.CanWrite) strokeProp.SetValue(ax.Setup, match.Stroke);
```

QMC.Common.Motion.AxisSetup 의 `DefaultVelocity` / `Stroke` 가 있으면 자동 반영. 없는 경우 무시 (best-effort).

## 4. Round 14: LightSetup RS-232C 명령 전송

SAVE 클릭 시 JSON 저장 + **모든 활성 채널에 RS-232C 명령 자동 전송**:

```
COM 포트별 그룹화 → SerialPort.Open(9600 8N1)
   각 채널: $LCH<CH>,<LEVEL>\r\n
```

**Sim 모드 처리**: COM 포트가 없거나 Open 실패 시 콘솔에 기록만:
```
[LIGHT-SIM] COM2 CH3 LEVEL=200 (Strobe)
```

결과 MessageBox: "조명 컨트롤러 명령 전송: N/8"

명령 형식은 메뉴얼 (CDT-300 / CDT-310 광원 컨트롤러 사양) 표준:
- `$LCH<채널>,<강도>` — 강도 설정
- `\r\n` 종료자

## 6. 사용자 시나리오 (Round 11-14 통합)

### 6.1 OutputStage Bin 위치 미세 조정

```
1. Position Teaching 페이지 → "OutputStage.BinCameraWorkPositionX" 행 선택
2. 값을 200.0 → 195.5 로 직접 편집
3. APPLY 클릭 → 즉시 m.OutputStage.Setup 에 반영
4. SAVE 클릭 → teach_positions.json 영속화
5. 다음 다이부터 새 위치 사용
```

### 6.2 카메라 일괄 진단

```
1. Camera Setup 페이지
2. TEST CONN 클릭
3. 6 채널 PORT 셀 색상 확인:
   - 5100 (Wafer)        🟢
   - 5101 (BottomInsp)   🟢
   - 5103 (Bin)          🟢
   - 5104 (Main)         🔴 ← Vision PC가 아직 안 켜짐
   - 5105 (FrontSide)      🔴
   - 5106 (RearSide)   🔴
4. 빨간 셀로 어느 채널이 끊겼는지 즉시 식별
```

### 6.3 조명 강도 즉시 조정

```
1. Light Setup 페이지 → CH2 BOTTOM VISION LEVEL 180 → 220 편집
2. SAVE 클릭
3. (실 RS-232C 연결 시) 자동으로 $LCH2,220\r\n 전송
4. 다음 사이클 부터 더 밝은 광원
```
