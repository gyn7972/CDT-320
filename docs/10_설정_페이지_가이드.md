# CDT-320 설정 페이지 가이드 (Stage 59)

**문서 버전**: 1.0
**작성일**: 2026-04-29
**대상**: 설정 탭의 신규 4개 페이지 + IO 매핑 편집 강화

---

## 1. 개요

핸들러 운영자가 시퀀스 위치 / 축 사양 / 카메라 / 조명을 GUI 에서 직접 편집할 수 있도록 4개 신규 설정 페이지가 추가되었다.

| 페이지 | 위치 | 저장 파일 |
|---|---|---|
| **Position Teaching (위치 티칭)** | 설정 탭 → 하단 그룹 | `Config/teach_positions.json` |
| **Axis Setup (축 셋업)** | 설정 탭 → 하단 그룹 | `Config/axis_setup.json` |
| **Camera Setup (카메라 셋업)** | 설정 탭 → 하단 그룹 | `Config/camera_setup.json` |
| **Light Setup (조명 셋업)** | 설정 탭 → 하단 그룹 | `Config/light_setup.json` |
| IO 리스트 (편집 가능) | 설정 탭 → DIGITAL/CYLINDER 등 | `Config/io_*.json` |

## 2. Position Teaching — 시퀀스 위치 티칭

### 2.1 항목 (32개)

각 시퀀스 위치를 **모듈별 그룹** 으로 표시:
- **InputLoader (2)**: FirstSlotPosition, ExchangePositionY
- **InputStage (8)**: ExpanderDown/Up, UnloadPositionY, NeedleEject/Down, PickerOffsetX/Y
- **TPU.Picker (4)**: Pick / Place / Focus / Wait Z 위치
- **TPU.Front (7)**: ArmInput/Inspect/Output X, Side1 X/Y, PickerPitchX, SideY0
- **TPU.Rear (4)**: ArmInput/Inspect/Output X, SideY0
- **OutputStage (5)**: StageBaseY, Work/Avoid Z, BinCamera Work/Retract X
- **OutputUnloader (7)**: NG/Good1/Good2 Slot 0 Z, SlotPitchZ, NG/Good 교환 Y, 카세트 삽입 Y

### 2.2 액션 버튼

- **TEACH** — 현재 행의 축의 ActualPosition 을 Value 로 적용
- **MOVE TO** — Value 위치로 해당 축 이동 (서보 자동 ON)
- **APPLY** — 모든 항목을 각 Unit 의 `Setup` 객체에 반영 (런타임 즉시 적용)
- **SAVE / RELOAD / RESET DEFAULT** — JSON 영속화

### 2.3 워크플로

```
1. 사용자가 핸들러 GUI 에서 축 jog 로 정확한 위치 도달
2. Position Teaching 페이지에서 해당 항목 행 선택
3. TEACH 버튼 → ActualPosition 자동 입력
4. SAVE → JSON 저장
5. 다음 사이클부터 그 위치 사용
```

## 3. Axis Setup — 37 축 사양 등록

### 3.1 컬럼

| 컬럼 | 의미 |
|---|---|
| NO | #00~#36 |
| MODULE | InputLoader / InputStage / FrontPicker / RearPicker / OutputStage / OutputUnloader |
| AXIS NAME | "WAFER LIFTER_Z" 등 메뉴얼 표준 명 |
| UNIT | mm / deg |
| STROKE | 메뉴얼 기준 max 이동 거리 |
| BRAKE | 0/7/36 = ON (브레이크 포함 축) |
| SOFT LIMIT(-) / SOFT LIMIT(+) | 안전 범위 |
| DEFAULT VEL | 기본 이동 속도 |
| HOME DIR | POS / NEG (기본 NEG) |

### 3.2 메뉴얼 기준 시드 (37 행)

코드: `AxisSetupPage.SeedDefault()` 참고. CDT-310/CDT-300 매뉴얼의 실측 값을 기반으로 구성:

```
#00 WAFER LIFTER_Z          200mm  BRK
#01 WAFER FEEDER_Y          300mm
#02 WAFER STAGE_Y           400mm
#03 WAFER STAGE_T           360deg
...
#19 FRONT SIDE VISION_Y0    200mm
#20 REAR SIDE VISION_Y0     200mm
...
#36 BIN LIFTER_Z            200mm  BRK
```

### 3.3 APPLY 버튼

`Setup.SoftLimitMinus / SoftLimitPlus` 를 실 축에 즉시 반영. 사이클 중에도 변경 가능 (단 안전 위해 사이클 정지 후 권장).

## 4. Camera Setup — Vision 6채널

### 4.1 채널

| Index | Channel | Role | Port |
|---|---|---|---|
| 0 | Wafer | WaferAlign | 5100 |
| 1 | BottomInspection | DiePresence | 5101 |
| 2 | Bin | PlacementInspector | 5103 |
| 3 | Main | MainComm | 5104 |
| 4 | FrontSide | FrontSide4Side | 5105 |
| 5 | RearSide | RearSide4Side | 5106 |

### 4.2 편집 가능 필드

- HOST / PORT — Vision PC 연결 정보
- WIDTH / HEIGHT — 해상도
- EXPOSURE(ms) — 노출 시간
- LIGHT — 광원 강도 (0.0~1.0)
- TRIGGER — Software / Hardware
- AUTO CONN — 시작 시 자동 연결 여부

### 4.3 액션

- **TEST CONN** — 선택 채널 TCP 연결 테스트 (2초 timeout)
- **APPLY (AppSettings)** — 6 포트를 `AppSettings.json` 에 반영 (재시작 후 적용)

## 6. IO 매핑 편집 (강화된 IoListPage)

기존 IoListPage 가 read-only 였으나 Stage 59 에서 **편집 가능** 으로 변경:
- ADD ROW / 행 삭제 가능
- SAVE → `Config/io_<카테고리>.json` 별도 파일에 영속화
- RELOAD → 디스크에서 다시 읽음

카테고리:
- DIGITAL (X / Y 통합 약 84개)
- DIGITAL LINK (인접 신호 매핑)
- CYLINDER (5개 — Feeder/NG Bin/Good Bin/Bottom Blow/Needle Vac)
- LAMP (7개)
- SWITCH (5개)
- LIGHT SOURCE (8개 — §5와 동일)

## 8. 사용 시나리오

### 8.1 신설 라인 셋업 흐름

1. **AxisSetup** → 37 축의 SoftLimit/Velocity/Brake 확인 (메뉴얼과 일치하는지)
2. **CameraSetup** → 6 채널의 IP/Port 확인 + TEST CONN 으로 연결 검증
3. **LightSetup** → 채널별 강도 조정 (촬상 결과 보면서 LEVEL 0~255 튜닝)
4. **PositionTeaching** → InputLoader.ExchangePositionY 부터 OutputUnloader 슬롯 위치까지 32 개 위치 티칭
5. **APPLY** 버튼들로 런타임 즉시 반영
6. **SAVE** 로 JSON 영속화
7. CYCLE RUN — 전체 사이클 검증

### 8.2 운영 중 미세 조정

- 다이 픽업 위치가 0.05mm 비뚤어졌을 때:
  1. PositionTeaching 페이지 → InputStage.PickerOffsetX/Y 행 선택
  2. 값을 직접 편집 또는 TEACH 로 현재 위치 적용
  3. APPLY → 다음 다이부터 즉시 적용
