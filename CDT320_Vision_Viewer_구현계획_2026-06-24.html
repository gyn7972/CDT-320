CDT-320 Vision 검사 결과 뷰어 — 신규 구현 계획

2026-06-24 · 대상: QMC.Vision (작업 모니터링 화면) · 참조 데이터모델: QMc.Vision.Inspector (CDT-310_Vision) · 프로토콜: Vision Protocol_20250529.xlsx
전제 — 스크린샷의 운영 뷰어 화면을 그리는 소스는 마운트된 어느 코드베이스(CDT-300/310/320)에도 없음. 따라서 신규 구현한다. 재사용 가능한 자산은 두 가지: ① QMc.Vision.Inspector의 검사 결과 클래스(데이터 모델), ② QMC.Vision의 OperationPage·CameraView·모듈/프레임 인프라(UI 골격).

0. 목표 요약 (듀얼 모니터)

  • 모니터 2개 구성. 위(2번) 모니터 = Bottom 검사 전용 풀스크린 고정 창, 아래(1번) 모니터 = 메인 프로그램 상주.
  • Bottom Inspection은 프로그램 실행 시 자동으로 위 모니터에 테두리 없는 풀스크린 새 창으로 띄우고 이동/리사이즈 잠금·상시 유지. 모드 전환 없음(Bottom 전용).
  • 아래 작업화면(작업 탭, 상시)의 메인 표시 영역은 우측에 세로로 배치한 3개 버튼(작업 · Side · Die)으로 전환된다. Bottom 버튼은 없음(Bottom은 위 모니터 전용 창이 담당).
  • "작업" 버튼 = 기존 작업 모니터 UI(현행 OperationPage 카드형: BOTTOM 메인 + 4모듈 2×2 + RUN/READY) — 기본 표시. Side · Die 버튼은 메인 영역을 측면/Bin 검사 화면으로 교체.

1. 아키텍처 — 데이터는 TCP가 아니라 로컬 결과 객체에 바인딩

프로토콜 분석 결과, 핸들러로 가는 TCP REPLY는 축약 데이터만 담는다:

  • BCI(Bottom, 5101) REPLY: Offset X/Y, Theta, Result Code(NG 코드 n개). → Width/Height·칩핑·이물 미포함
  • OCI(Output/Bin, 5103) REPLY: Offset X/Y, Rotate Angle, Top/Left/Bottom/Right Gap의 Min/Max.
  • SCI(Side) REPLY: 판정 중심.

반면 스크린샷의 화면은 너비/높이 추세, ChippingSize Map, 이물, per-die 결과표까지 보여준다 → 이는 비전 PC 로컬 검사 결과 객체(BottomResult 등)를 직접 렌더한 것. 비전 PC가 곧 검사 주체이므로, 뷰어는 로컬 결과 스토어에 바인딩한다.

기존 ModuleResultStore는 판정 + 텍스트 라인만 보관(per-die 구조화 없음). 추세 그래프·결과 그리드·Map을 위해 구조화 per-die 스토어를 신설한다.

3-1. 듀얼 모니터 — Bottom 전용 풀스크린 창 신규 핵심

+ Ui\Windows\BottomInspectionWindow.cs / .Designer.cs
  • Form 파생. FormBorderStyle = None, StartPosition = Manual, ShowInTaskbar = false, ControlBox = false.
  • 대상 모니터 선택: Screen.AllScreens에서 보조(비-Primary) 또는 Primary 위쪽(Bounds.Y < primary.Y) 화면 선택. 인덱스는 AppSettings로 설정 가능, 단일 모니터면 Primary 폴백.
  • 꽉 채움: Bounds = targetScreen.Bounds 직접 지정(테두리 없음 → 작업표시줄까지 덮음). TopMost = true로 상시 전면.
  • 이동/리사이즈 잠금: 테두리 None으로 기본 차단 + WndProc에서 WM_SYSCOMMAND의 SC_MOVE/SC_SIZE 무시. 닫힘 방지(FormClosing에서 종료 외 취소) — 메인 종료 시에만 닫힘.
  • 자동 기동: Form1(메인) 초기화 후 Show(). 디스플레이 변경(SystemEvents.DisplaySettingsChanged) 시 위치 재적용.
  • 내용 = Bottom 모드 뷰어 콘텐츠(§4 공통 컨트롤 재사용). InspectionResultStore(Bottom) 구독.

4. 기능별 구현 항목

4-1. 구조화 결과 스토어 신규

+ Equipment\Core\InspectionResultStore.cs
  • 모드(Bottom/Side/Bin)별 per-die 레코드 리스트 보관: {IndexX, IndexY, Picker, 측정값..., DefectCode, Time}.
  • Record(mode, record) + ResultsAdded 이벤트(리비전 증가). 뷰어가 구독하여 그래프/그리드/Map 갱신.
  • Lot 시작 시 Clear(mode). 최신 N개(예: 250) 링버퍼로 추세 그래프 X축 구성.
  • 결과 객체는 Inspector 어댑터(BCI/SCI/OCI 처리부)에서 TCP REPLY 직전에 push.

4-2. 공유 뷰어 콘텐츠 컨트롤 + 호스팅 신규

+ Ui\Controls\InspectionViewerControl.cs / .Designer.cs  (UserControl, 공통 프레임)
+ 작업화면(WorkTab/OperationPage) 우측 세로 버튼바(작업·Side·Die) + 메인 영역 전환
  • 핵심 분리: 검사 화면 콘텐츠(헤더·Picker·그래프·Map·그리드)는 InspectionViewerControl(UserControl) 하나로 만들고, 위 모니터 창(BottomInspectionWindow = Bottom 고정)과 작업화면 메인 영역(Side/Bin)이 각각 인스턴스를 호스팅.
  • 작업화면 우측 세로 버튼바: 작업 탭 우측에 고정폭 컬럼(FlowLayoutPanel 세로 또는 TableLayoutPanel 1열 3행), 버튼 작업 · Side · Die. Bottom 버튼 없음.
  • 메인 영역 전환: "작업" = 기존 OperationPage 모니터 콘텐츠(카드형) 그대로 표시(기본). "Side"/"Die" = 같은 영역에 InspectionViewerControl(SetMode(Side/Bin)) 표시. 같은 호스트 패널에서 자식 컨트롤만 교체(Visible/BringToFront).
  • Designer 규칙(AGENTS.md): 컨트롤은 .Designer.cs 인라인, 로직 .cs. partial·클래스명=파일명. 버튼 이벤트 btnWork_Click/btnSide_Click/btnDie_Click, 활성 버튼 하이라이트.
  • SetMode(InspectionMode) 하나로 라벨·패널·그래프 시리즈·그리드 컬럼·Map 표시여부를 모드 데이터로 스왑(목업의 sel()과 동일 개념). 위 창은 SetMode(Bottom) 고정.

4-3. Picker 1~4 패널 재사용

  • 기존 CameraView 재사용: SetFrame(GrabResult)(이미지), SetVerdict("OK/NG", pass)(Good/NG), SetResultLines(string[])(W/H·gap 텍스트), SetOverlay(roi, marks)(녹색 박스/코너).
  • 이미지 소스: BottomResult.DisplayImage 직접 or 기존 IVisionModule.AcquireViewerFrame() 비침습 탭(ViewerFrameSeq) 경로.
  • Picker index → 패널 인덱스 매핑(1~4).

4-4. 추세 그래프(SPC) ①② 신규 컨트롤

  • X=die 순번, Y=측정값, UpperLimit/LowerLimit 점선(레시피 규격). 모드별 시리즈: 너비·높이 / Front·Back chipping depth / Right·Bottom gap.
  • 구현 옵션 — (A) System.Windows.Forms.DataVisualization.Charting.Chart(MS Chart, .NET FW 4.7.2 내장) 권장, (B) 경량 커스텀 Panel.OnPaint 폴리라인. 규격선은 레시피의 상/하한에서 가져옴(RecipeStore/MaterialSpecs).
  • InspectionResultStore.ResultsAdded 구독 시 최신 윈도우로 다시 그림.

4-5. Map(Bottom 전용) 신규

  • Width/Height/1ch/2ch ChippingSize를 die 격자에 히트맵(값↑=진한 빨강) 표시. Panel.OnPaint 격자 렌더.
  • 좌표=die IndexX/IndexY, 값=Channel1/2ChippingSize 등.

4-6. 결과 그리드 재사용

  • DataGridView(또는 기존 SortableBindingList/GridTheme) — 모드별 컬럼셋. InspectionResultStore 레코드 바인딩.
  • 최신 행이 위로(역순). 컬럼은 §3 표 기준.

4-7. 헤더 · 토글 · 하단 메뉴 수정

  • 헤더: 레시피명/Lot ID/Wafer ID = 현재 Lot 컨텍스트에서. Judge/Normal = 누적 판정.
  • 상단 토글: 모드별 라벨(너비와높이 / Channel1·2 / 위·좌·아래·우) — 보기 옵션 전환(그래프 대상 채널/방향 선택).
  • 하단 메뉴(작업/구성/조작/레시피/데이터로그)는 기존 탭 셸(WorkTab 등) 재사용.

5. 데이터 흐름(시퀀스)

  1. 핸들러 → 비전: BCI/SCI/OCI REQUEST (STX TID,REQUEST,CMD,cnt,args ETX).
  2. Inspector 실행 → BottomResult/SideResult/DieGapResult 생성.
  3. 어댑터: ① InspectionResultStore.Record() (전체값) → ② TCP REPLY(축약값) 핸들러 전송.
  4. 뷰어: ResultsAdded 수신 → 현재 모드면 Picker/그래프/Map/그리드 갱신. UI 스레드 마샬링(BeginInvoke), 비침습.

7. 규칙 준수 체크 (AGENTS.md)

  • Designer: 컨트롤은 .Designer.cs 인라인, 로직은 .cs. partial·클래스명=파일명.
  • 예외: try/catch/finally, catch{} 금지. 실패 시 로그(+UI면 메시지박스).

8. 진행 순서(제안)

  1. InspectionResultStore + 어댑터 push (데이터 파이프라인 먼저).
  2. InspectionViewerControl 골격 + Bottom 모드 풀구현(헤더·Picker·너비/높이 그래프·Map·그리드).
  3. BottomInspectionWindow(위 모니터 풀스크린 고정) + Form1 자동 기동 → 실행 시 상시 표시·보조모니터 배치 확인.
  4. SideBinViewerPage(아래 메인) + Side/Bin 버튼 전환 + Side·Bin 콘텐츠.
  5. SPC 상/하한 = 레시피 규격 연동, 판정/색상 마감.
  6. VS2022 빌드 → 5모듈 결과 연동 + 듀얼모니터 배치 테스트(perl tools/verify_vision_features.pl).
